Extracty in a nutshell

Meet John.

He is a software developer and he needs to extract data from the Internet.

In todayâ€™s world, the Internet offers nearly unlimited amount of valuable data. The problem is that data is often unstructured and spread over different Internet sources.

Most online data is accessible through a web browser but less than 1% of it is accessible via an API.

Most webpages design the presentation of the data in a way that is easily understandable by humans but much more difficult to process and consume for machines and software systems. Extracting this data in a structured and scalable way has long been a challenge for developers.

John needs his data in JSON and structured in exactly the way he needs it.

The data available on the web is noisy and often contains portions which are irrelevant to the developer's needs. In many cases, extra calculations, filtering and/or more complex processing of the data is needed before it can be used. Using Extracty, John can apply any programmable logic during the extraction process to get exactly the data and the format he needs.

After the setup, he can access the data through a simple GET request and can even pass parameters which effect the extraction logic.

Extracty endpoints can consume parameters which could define the pages the data should be extracted from, filters that should be applied or any other logic that takes dynamic input to compute the desired output.

John can finally build his shiny new app without having to develop his own back-end server or database.

Using Extracty, he can create an API that extracts and returns exactly the web data he needs and use it directly in his new app.
